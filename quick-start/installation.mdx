Get the Guru Network Framework running locally in just a few minutes with our streamlined installation process.

## üê≥ Docker Installation (Recommended)

### Step 1: Clone Repository
```bash
git clone <repository-url>
cd guru-network-mono
```

### Step 2: Environment Setup
```bash
# Copy environment template
cp .env.example .env

# Edit configuration (optional - defaults work for local development)
nano .env
```

### Step 3: Start Services
```bash
# Start core services (engine, gui, workers)
docker-compose up -d --build

# Or start with logs visible
docker-compose up --build
```

### Step 4: Verify Installation
<Steps>
  <Step title="Check Service Status">
    ```bash
    docker-compose ps
    ```
    All services should show "Up" status
  </Step>
  <Step title="Access Web Interface">
    - **Frontend**: http://localhost:3000 (Next.js Telegram Mini App)
    - **Camunda Cockpit**: http://localhost:8080/camunda (demo/demo)
    - **Workers**: Check status with `docker-compose logs external-workers`
  </Step>
  <Step title="Test Services">
    ```bash
    # Check Camunda engine
    curl http://localhost:8080/engine-rest/engine

    # Check frontend
    curl http://localhost:3000
    ```
  </Step>
</Steps>

## üö® Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Docker Compose Fails to Start">
    **Symptoms**: Services fail to start or show error status

    **Solutions**:
    ```bash
    # Check Docker daemon is running
    docker version

    # Check port conflicts
    netstat -tlnp | grep :8080  # Camunda engine
    netstat -tlnp | grep :3000  # Frontend

    # Reset Docker Compose
    docker-compose down -v
    docker-compose up -d --build --force-recreate
    ```
  </Accordion>

  <Accordion title="Worker Connection Issues">
    **Symptoms**: "Connection refused" to Camunda engine

    **Solutions**:
    ```bash
    # Check worker logs
    docker-compose logs external-workers

    # Verify engine is accessible
    curl http://localhost:8080/engine-rest/engine

    # Restart workers
    docker-compose restart external-workers
    ```
  </Accordion>
</AccordionGroup>
